Dialog Component API

The Dialog is an abstraction of the Clarity default Modal

Dialog Component Attributes

A Dialog can be initialized with any of the following options from model ModalConfigurationModel:
data any
The data to be passed to the Dialog to be used in the component
title string
Defined the Title of the Dialog, at this point it can't be change after initialized
modalSize ModalSize
Determinate the Size of the Modal. Values: "sm", "md" "lg" "xl"
draggable boolean (False)
Allows the Dialog to be Draggable from the Header
resizable boolean (False)
Allows the be Resizable from the borders: top, right, bottom, left, top-right, top-left, bottom-right, bottom-left
fullScreen boolean (False)
Enables the buttons to be able to Maximize and Minimize a Dialog
defaultFullScreen boolean (False)
By Default init the dialog in full screen and also enables the buttons to be able to Maximize and Minimize a Dialog
showBackground boolean (False)
By Default it Shows the background (Gray Layout) unless is passed as false. Also even disabled, user can't interact with anything behind the dialog.

Dialog Buttons Implementation

Beside of the options, every component is in charge of describe the behavior by extending the Dialog class. Buttons are models from DialogButtonModel described as:
type DialogButtonType
it can be DialogButtonType.ACTION (shows on the left panel) or DialogButtonType.CONTEXT shows at the bottom
name string
Inner Button name that can be referenced to apply any change later, like hide or show
text string
The label text of the button that applies only to type DialogButtonType.CONTEXT
icon string
Icon to be used following Clarity Icons or Font-awesomeshow
show boolean
Show or hide the button, it can be change at running time.
disabled boolean
Disable the button, it can be change at running time.
action Function
Returns a callback after the button event propagate on click

Dialog Events

nodeClicked EventEmitter
Event emitted when a node is clicked.
